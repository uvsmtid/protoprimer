Files contain references formatted as:

```
PREFIX_NN_NN_NN_NN.semantic_name.md
```

For example:

```
UC_44_35_26_17.some_requirement.md
FT_08_17_26_35.some_implementation.md
TODO_41_52_64_75.some_task.md
```

The referenced files contain the background relevant to the place they appear.

To find and read the doc, use the `PREFIX_NN_NN_NN_NN` part (as semantic name may diverge):

*   `UC_NN_NN_NN_NN` -> `./doc/use_case`
*   `FT_NN_NN_NN_NN` -> `./doc/feature_topic`
*   `TODO_41_52_64_75` -> `./doc/task_ref`
