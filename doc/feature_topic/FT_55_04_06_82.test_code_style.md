# FT_55_04_06_82.test_code_style

## test code extends prod code style

Apply rules from [prod_code_style][FT_43_27_70_50.prod_code_style.md].

## given when then

When writing tests, use markers: given, when, then.

Put comments (if any) after on a separate (next) line after the marker.
Separate each marker by an empty line before and after.

```py

# given:

# when:

# then:

```

## `pytest`-style

Use `pytest`-style tests without test classes.

## use @patch for mocks

Instead of using `with` to define mocks, for example:

```py
with patch("sys.argv", [""]):
    pass
```

use `@patch` decorator at the function definition for mocks:

```py
@patch("sys.argv", [""])
def test_function():
    pass
```
